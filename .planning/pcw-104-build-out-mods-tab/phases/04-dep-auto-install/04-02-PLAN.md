Purpose: Complete the dep auto-install UX so users never hit a "missing dependency" crash.
Output: VersionPickerModal handles deps transparently; JourneyMap installs with Fabric API.
</objective>

<task type="auto">
  <name>Task 1: Add dep-check state and confirmation UI to VersionPickerModal</name>
  <files>src/renderer/components/mods/VersionPickerModal.tsx</files>
  <action>
    Add the following state to the component:
    - `pendingDeps: DependencyInfo[] | null` — null = not checked yet; [] = no deps needed
    - `depCheckLoading: boolean` — true while getRequiredDeps is in flight

    Change `handleInstall` to a two-phase flow:

    **Phase 1 — dep check (first call):**
    If `pendingDeps === null`:
    1. Set `depCheckLoading = true`
    2. Call `window.launcher.mods.getRequiredDeps(instanceId, selectedVersionId)`
    3. If result is empty array: skip to Phase 2 immediately (no deps needed)
    4. If result is non-empty: set `pendingDeps = result`, `depCheckLoading = false`, return
       (UI now shows confirmation prompt — Install button becomes "Install all (N+1 mods)")

    **Phase 2 — actual install (second call, or immediate if no deps):**
    `pendingDeps` is now either `[]` or confirmed by user:
    - Sequential install: for each dep in `pendingDeps`: call `mods.install(instanceId, dep.project_id, dep.versionId, dep.title, dep.slug)`
    - Then install the originally selected mod (existing logic)
    - Set `installing = false` only after all complete; call `onInstalled()`

    **Confirmation UI** (rendered when `pendingDeps !== null && pendingDeps.length > 0`):
    Replace the version selector area with:
    ```
    <p>This mod requires:</p>
    <ul>{pendingDeps.map(d => <li key={d.project_id}>{d.title} ({d.versionNumber})</li>)}</ul>
    <p className="form-group__hint">These will be installed automatically.</p>
    ```
    The Install button label becomes: `Install all ({pendingDeps.length + 1} mods)`

    **Status label during install** (when `installing === true`):
    Show `installingLabel` state string below the button (e.g. "Installing Fabric API…",
    "Installing JourneyMap…"). Update it before each `mods.install` call.

    Do NOT add a "Skip dependencies" option — required deps are required.
    Do NOT change the Cancel button behavior.
  </action>
  <verify>
    In dev: open VersionPickerModal on a mod with no deps → single click installs as before.
    On a mod with required deps → first click shows dep list, second click installs all.
  </verify>
  <done>
    Two-phase install flow working; dep list rendered when deps found;
    Install button label reflects total mod count; status label updates during install.
  </done>
</task>
