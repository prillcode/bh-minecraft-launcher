<objective>
Add a `mods:get-required-deps` IPC handler that, given an instance and a selected mod version ID,
returns the list of required dependencies that are not yet installed — with enough info (project_id,
slug, title, versionId, versionNumber) for the renderer to display a confirmation prompt and install
each dep via the existing `mods:install` handler.

Purpose: Provide the data layer for dep auto-install without changing existing install flow.
Output: New IPC channel wired end-to-end; `DependencyInfo` type available in renderer.
</objective>

<task type="auto">
  <name>Task 2: Add mods:get-required-deps IPC handler</name>
  <files>src/main/ipc-handlers.ts</files>
  <action>
    Add a new handler after the existing `mods:install` handler:

    ```
    ipcMain.handle('mods:get-required-deps', async (_e, instanceId: string, versionId: string) => { ... })
    ```

    Logic:
    1. Fetch the selected version: `modrinthAPI.getVersion(versionId)` — already has dependencies[]
    2. Filter to `dependency_type === 'required'`
    3. Get already-installed mod IDs: `modManager.list(instanceId)` → extract `id` (project_id) set
    4. For each required dep NOT in the installed set:
       a. Resolve the install version ID:
          - If `dep.version_id` is set: use it directly
          - Otherwise: call `modrinthAPI.getVersions(dep.project_id, { gameVersions: [instance.versionId], loaders: [loaderArr] })` and take `[0].id` (latest compatible). If no compatible version found, skip this dep (log a warning).
       b. Get the project info for display: `modrinthAPI.getProject(dep.project_id)` → title, slug
       c. Also fetch `modrinthAPI.getVersion(resolvedVersionId)` to get `version_number`
    5. Return array of `{ project_id, slug, title, versionId: resolvedVersionId, versionNumber }`

    For `loaderArr`: derive from `instance.modLoader` — if falsy or 'vanilla' pass undefined (no loader filter).

    Note: `modrinthAPI` and `modManager` and `instanceManager` are already in scope in `registerIpcHandlers`.
  </action>
  <verify>
    In dev mode: open DevTools console, call
    `window.launcher.mods.getRequiredDeps(instanceId, knownVersionIdWithDeps)` — returns array or [].
    TypeScript compiles without errors.
  </verify>
  <done>
    Handler registered; returns [] for mods with no required deps; returns populated array
    for mods that have uninstalled required deps.
  </done>
</task>
