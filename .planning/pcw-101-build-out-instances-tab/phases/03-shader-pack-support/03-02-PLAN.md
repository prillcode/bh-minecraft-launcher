<task type="auto">
  <name>Task 2: ShaderVersionPickerModal component</name>
  <files>src/renderer/components/shaders/ShaderVersionPickerModal.tsx</files>
  <action>
    Create `ShaderVersionPickerModal.tsx`. Mirrors `VersionPickerModal` but simpler:
    no dependency check, no dep state.

    Props:
    ```ts
    interface Props {
      shader: ModSearchHit;       // reuse existing ModSearchHit type — shader search hits have same shape
      instanceId: string;
      instances: InstanceInfo[];
      onClose: () => void;
      onInstalled: (shader: ShaderInfo) => void;
    }
    ```

    On mount: call `window.launcher.mods.getVersions(shader.slug, instance.versionId, undefined)`
    to fetch versions (no loader filter — shader packs aren't loader-specific).
    Store as `versions: ModVersionInfo[]`.

    State: `selectedVersionId`, `loading`, `installing`, `downloadPercent: number`, `error`.

    Install flow (single click, no dep check):
    1. Set `installing = true`
    2. Subscribe to download progress:
       ```ts
       const unsub = window.launcher.shaders.onDownloadProgress(({ percent }) => {
         setDownloadPercent(percent);
       });
       ```
    3. Call `window.launcher.shaders.installModrinth(instanceId, selectedVersionId, shader.title)`
    4. Call `unsub()` to remove listener
    5. Call `onInstalled(result)`

    Show download progress as a label: "Downloading… 42%" when `installing && downloadPercent > 0`.
    Show file size hint below version selector (same as VersionPickerModal).

    Overlay click and Escape key to close (same pattern as VersionPickerModal).
    Use existing modal CSS classes: `modal-overlay`, `modal`, `modal__header`, `modal__title`,
    `modal__close`, `modal__actions`, `form-group`.
  </action>
  <verify>File created. `pnpm build` passes with no TS errors.</verify>
  <done>
    ShaderVersionPickerModal renders version list, shows download progress,
    calls onInstalled with ShaderInfo on success.
  </done>
</task>

<task type="auto">
  <name>Task 3: ShadersTab + App router + Sidebar nav</name>

  <action>
    **ShadersTab.tsx** — mirrors ModsTab structure:

    State: `instances`, `selectedInstanceId`, `query`, `results: ModSearchHit[]`,
    `searching`, `searchError`, `shaderPickerMod: ModSearchHit | null`,
    `installedRefreshKey`, `localInstalling`.

    `selectedInstance` derived from instances + selectedInstanceId.
    `isSupported` = `selectedInstance?.modLoader === 'fabric' || selectedInstance?.modLoader === 'quilt'`
    (gate — vanilla and missing modLoader are unsupported).

    On mount: load instances list, auto-select first.

    Instance selector onChange: clear results, increment refreshKey.

    Search: calls `window.launcher.mods.search(query, selectedInstanceId)` with
    `project_type: 'shader'` — wait, `mods.search` passes instanceId and the handler
    already applies loader filter. For shaders we don't want a loader filter. Check
    ipc-handlers.ts: `mods:search` applies `loader` only if `instance.modLoader !== 'vanilla'`.
    Shader packs are NOT loader-specific, so to avoid filtering by loader, call the search IPC
    with a special approach: since we can't change mods:search easily, use
    `window.launcher.mods.search(query + ' ', selectedInstanceId)` — actually, the cleanest
    approach is to search via the existing `mods:search` handler but the loader filter will
    be applied. For Fabric instances this is fine since Iris (Fabric) shader packs are listed
    under `fabric` loader on Modrinth. Accept this behaviour — don't add a new search handler.

    Search results: render shader cards identical to mod cards in ModsTab (icon, title, desc,
    downloads, Install button). Clicking Install opens `ShaderVersionPickerModal`.

    "Install from file" button: calls `window.launcher.shaders.installLocal(selectedInstanceId)`,
    on non-null result increments `installedRefreshKey`. Show loading state on button while in flight.

    Gate: when `!isSupported` show message:
    "Shader packs require a mod loader that supports Iris (Fabric or Quilt).
    Edit the instance and switch the Mod Loader to Fabric to enable shader support."
    Disable search input, search button, and install-from-file button when `!isSupported`.

    Below results: render `<InstalledShadersPanel instanceId={selectedInstanceId} refreshKey={installedRefreshKey} />`
    (only when `isSupported && selectedInstanceId`).

    **App.tsx** — add import and route:
    ```tsx
    import { ShadersTab } from './components/shaders/ShadersTab';
    // in Routes:
    <Route path="/shaders" element={<ShadersTab />} />
    ```

    **Sidebar.tsx** — add Shaders nav item between Mods and Settings:
    ```ts
    { to: '/shaders', label: 'Shaders', icon: '✨' },
    ```
