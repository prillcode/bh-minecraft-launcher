# Plan 01-02: Create Instance Modal + Upgrade Instance List UI

## Objective
Build the CreateInstanceModal component (name, version picker, optional server) and upgrade the existing InstanceList with delete buttons, a better empty state, and real typed data. Add all needed CSS.

## Context
- @src/renderer/components/instances/InstanceList.tsx — Existing skeleton: lists instances, has play button, has "New Instance" button with TODO onClick
- @src/renderer/global.d.ts — `InstanceConfig`, `InstanceInfo` types (from plan 01-01), plus `game.getVersions()` returning `{ latest, versions[] }`
- @src/renderer/styles/globals.css — Existing `.instance-card`, `.instances__*`, `.btn`, `.btn--primary` styles. CSS vars: `--bg-primary/secondary/tertiary/elevated`, `--border`, `--border-subtle`, `--text-primary/secondary/muted`, `--accent`, `--accent-hover`, `--accent-dim`, `--danger`, `--radius-sm/md/lg`, `--font-sans/mono`
- @src/renderer/App.tsx — Routes to `/instances` with `<InstanceList />`

## Tasks

### Task 1: Create the CreateInstanceModal component
- **Type:** code
- **Files:** `src/renderer/components/instances/CreateInstanceModal.tsx` (new file)
- **Action:**
  1. Create a modal overlay component that accepts `onClose: () => void` and `onCreate: (instance: InstanceInfo) => void` props
  2. State: `name` (string), `versionId` (string), `serverHost` (string), `serverPort` (string), `versions` (array from API), `loadingVersions` (bool), `creating` (bool), `error` (string|null)
  3. On mount, fetch versions via `window.launcher.game.getVersions()` — filter to `type === 'release'` only, populate a `<select>` dropdown, default to `latest.release`
  4. Form fields:
     - Instance Name: text input (required, placeholder "My Instance")
     - Version: `<select>` dropdown of release versions
     - Server section (collapsible or always visible with subtle label "Auto-connect to server (optional)"):
       - Server Host: text input (placeholder "e.g. mc.example.com")
       - Server Port: text input (placeholder "25565", type="number")
  5. "Create" button: validate name is non-empty, call `window.launcher.instances.create(config)` where config is `{ name, versionId, serverAutoConnect }` (only include serverAutoConnect if host is non-empty), call `onCreate` with the result, call `onClose`
  6. Handle errors inline
  7. Click on overlay backdrop (outside modal) calls `onClose`
  8. Escape key calls `onClose`
- **Verify:** File compiles — check with `npx tsc -p tsconfig.renderer.json --noEmit`

### Task 2: Upgrade InstanceList component
- **Type:** code
- **Files:** `src/renderer/components/instances/InstanceList.tsx`
- **Action:**
  1. Import `CreateInstanceModal` and the `InstanceInfo` type (or use inline from global.d.ts)
  2. Add `showCreateModal` state (boolean)
  3. Wire "New Instance" button to `setShowCreateModal(true)`
  4. When modal's `onCreate` fires, prepend the new instance to the list (or re-fetch)
  5. Add a delete button to each instance card (small X or trash icon) — calls `window.launcher.instances.delete(id)` with confirmation via `confirm()`, then removes from state
  6. Use the `InstanceInfo` type instead of the local `Instance` interface
  7. Show `serverAutoConnect.host` in the card if present (subtle text like "Server: mc.example.com")
  8. Better empty state: show a message and a "Create your first instance" button that opens the modal
  9. Render the modal when `showCreateModal` is true
- **Verify:** `npx tsc -p tsconfig.renderer.json --noEmit` passes
