# Plan 01-01: Wire Instance IPC Handlers + Strengthen Types

## Objective
Connect the existing `InstanceManager` class to the Electron IPC layer so the renderer can list, create, and delete instances. Also tighten the `global.d.ts` types for `instances.create()` so the renderer knows the exact shape.

## Context
- @src/core/game/instance.ts — `InstanceManager` with `list()`, `create()`, `get()`, `delete()`, `markPlayed()`
- @src/core/game/types.ts — `Instance` interface (id, name, versionId, modLoader, serverAutoConnect, etc.)
- @src/main/ipc-handlers.ts — IPC handlers; instances:* channels exist in preload but are NOT wired to InstanceManager yet
- @src/main/preload.ts — `instances.list/create/update/delete` already exposed via ipcRenderer
- @src/renderer/global.d.ts — Type declarations for `window.launcher`; `instances.create` currently typed as `(config: unknown) => Promise<unknown>`

## Tasks

### Task 1: Wire InstanceManager into IPC handlers
- **Type:** code
- **Files:** `src/main/ipc-handlers.ts`
- **Action:**
  1. Import `InstanceManager` from `../core/game/instance`
  2. Instantiate `const instanceManager = new InstanceManager();` alongside the other singletons at the top
  3. Add handler `instances:list`: call `instanceManager.list()`, return the array
  4. Add handler `instances:create`: receives `(_event, config)` — call `instanceManager.create(config)`, return the created instance
  5. Add handler `instances:delete`: receives `(_event, id: string)` — call `instanceManager.delete(id)`, return `{ success: true }`
- **Verify:** `npx tsc -p tsconfig.main.json --noEmit` passes with 0 errors

### Task 2: Tighten global.d.ts types for instances
- **Type:** code
- **Files:** `src/renderer/global.d.ts`
- **Action:**
  1. Add an `InstanceConfig` interface for the create form input:
     ```ts
     interface InstanceConfig {
       name: string;
       versionId: string;
       serverAutoConnect?: { host: string; port: number };
     }
     ```
  2. Add an `InstanceInfo` interface for what's returned:
     ```ts
     interface InstanceInfo {
       id: string;
       name: string;
       versionId: string;
       modLoader?: string;
       lastPlayed?: number;
       serverAutoConnect?: { host: string; port: number };
       createdAt: number;
     }
     ```
  3. Update the `instances` section of `LauncherAPI`:
     ```ts
     instances: {
       list(): Promise<InstanceInfo[]>;
       create(config: InstanceConfig): Promise<InstanceInfo>;
       update(id: string, config: Partial<InstanceConfig>): Promise<InstanceInfo>;
       delete(id: string): Promise<{ success: boolean }>;
     };
     ```
- **Verify:** `npx tsc -p tsconfig.renderer.json --noEmit` passes with 0 errors

### Task 3: Verify both tsconfigs compile cleanly
- **Type:** verify
- **Action:** Run both `npx tsc -p tsconfig.main.json --noEmit` and `npx tsc -p tsconfig.renderer.json --noEmit`
- **Verify:** Both pass with 0 errors

## Verification
- [ ] `instances:list` IPC handler calls `instanceManager.list()` and returns array
- [ ] `instances:create` IPC handler accepts config, creates instance, returns it
- [ ] `instances:delete` IPC handler accepts id, deletes instance, returns success
- [ ] `global.d.ts` has typed `InstanceConfig` and `InstanceInfo` interfaces
- [ ] Both tsconfigs compile with 0 errors
