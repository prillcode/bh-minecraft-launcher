<objective>
Update ModsTab and ShadersTab to consume selectedInstanceId from context instead of
maintaining their own instance dropdowns. Replace the dropdown with a compact
"Instance: [name] (version)" header label and a "← Change" link that navigates
to /instances. Vanilla instances are naturally excluded because the gate message
handles them — no dropdown means no way to accidentally select one here.

Purpose: Tabs no longer have redundant instance selection UI — they reflect whatever
the user selected on the Instances tab.
Output: Both tabs read from context; instance dropdown removed; header label + Change link in place.
</objective>

<task type="auto">
  <name>Task 1: Update ModsTab to use context</name>
  <files>src/renderer/components/mods/ModsTab.tsx</files>
  <action>
    Import context and router hooks:
    ```tsx
    import { useSelectedInstance } from '../../stores/selected-instance-context';
    import { useNavigate } from 'react-router-dom';
    ```

    Replace the existing `selectedInstanceId` + `setSelectedInstanceId` state with context:
    ```tsx
    const { selectedInstanceId } = useSelectedInstance();
    const navigate = useNavigate();
    ```

    Keep the `instances` state + `useEffect` to load the list (still needed to pass to
    `VersionPickerModal` and to look up the selected instance's details):
    ```tsx
    const [instances, setInstances] = useState<InstanceInfo[]>([]);
    useEffect(() => {
      window.launcher.instances.list().then(setInstances);
    }, []);
    ```

    `selectedInstance` and `isVanilla` derivations remain unchanged.

    **Replace the instance selector dropdown** in the JSX with a compact header:
    ```tsx
    <div className="mods__instance-header">
      {selectedInstance ? (
        <>
          <span className="mods__instance-label">
            Instance: <strong>{selectedInstance.name}</strong> ({selectedInstance.versionId})
          </span>
          <button className="btn btn--ghost btn--sm" onClick={() => navigate('/instances')}>
            ← Change
          </button>
        </>
      ) : (
        <>
          <span className="mods__instance-label mods__instance-label--none">No instance selected</span>
          <button className="btn btn--ghost btn--sm" onClick={() => navigate('/instances')}>
            ← Select Instance
          </button>
        </>
      )}
    </div>
    ```

    Remove the `<select>` element and its `onChange` handler entirely.
    The `installedRefreshKey` increment on instance change is no longer triggered by a
    dropdown — instead, add a `useEffect` that increments `installedRefreshKey` whenever
    `selectedInstanceId` changes:
    ```tsx
    useEffect(() => {
      setResults([]);
      setInstalledRefreshKey((k) => k + 1);
    }, [selectedInstanceId]);
    ```
    (Remove the inline `setResults([])` + `setInstalledRefreshKey` from the old onChange.)
  </action>
  <verify>
    `pnpm build` passes. ModsTab no longer has a `<select>` for instances.
    Header label + Change button present in JSX.
  </verify>
  <done>
    ModsTab reads selectedInstanceId from context. Instance dropdown replaced with
    label + Change navigation button.
  </done>
</task>
