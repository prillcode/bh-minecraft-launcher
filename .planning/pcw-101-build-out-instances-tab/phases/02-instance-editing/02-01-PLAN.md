# Plan 02-01: Wire instances:update IPC + Build EditInstanceModal

## Objective
Wire the `instances:update` IPC handler and build an `EditInstanceModal` component that pre-populates with the existing instance's data (name, version, server). Add an edit button to each instance card in `InstanceList` that opens the modal. On save, update the card in-place.

## Context
- @src/main/ipc-handlers.ts — `instances:update` channel is exposed in preload but has no handler yet; `instanceManager` singleton already exists in this file
- @src/core/game/instance.ts — `InstanceManager.update(id, Partial<Instance>)` returns the updated `Instance`
- @src/renderer/components/instances/InstanceList.tsx — has instance cards with play + delete buttons; needs edit button
- @src/renderer/components/instances/CreateInstanceModal.tsx — reference for modal structure and form logic
- @src/renderer/global.d.ts — `InstanceInfo`, `InstanceConfig` already declared globally; `instances.update()` typed as `(id, Partial<InstanceConfig>) => Promise<InstanceInfo>`
- @src/renderer/styles/globals.css — `.modal-overlay`, `.modal`, `.form-group`, `.form-row`, `.btn--ghost` already defined; no new CSS needed

## Tasks

### Task 1: Wire instances:update IPC handler
- **Type:** code
- **Files:** `src/main/ipc-handlers.ts`
- **Action:**
  1. Add handler `instances:update`: receives `(_event, id: string, config)` — call `instanceManager.update(id, config)`, return the updated instance
- **Verify:** `npx tsc -p tsconfig.main.json --noEmit` passes with 0 errors

### Task 2: Create EditInstanceModal component
- **Type:** code
- **Files:** `src/renderer/components/instances/EditInstanceModal.tsx` (new file)
- **Action:**
  1. Accept props: `instance: InstanceInfo`, `onClose: () => void`, `onUpdate: (updated: InstanceInfo) => void`
  2. Initialize state from the existing instance: `name` = `instance.name`, `versionId` = `instance.versionId`, `serverHost` = `instance.serverAutoConnect?.host ?? ''`, `serverPort` = `String(instance.serverAutoConnect?.port ?? 25565)`
  3. Fetch versions via `window.launcher.game.getVersions()` on mount (same as CreateInstanceModal) — filter to releases only
  4. Form fields (identical layout to CreateInstanceModal):
     - Instance Name: text input
     - Version: `<select>` dropdown
     - Server section: Host + Port side by side in `.form-row`
  5. "Save" button: validate name non-empty, call `window.launcher.instances.update(instance.id, { name, versionId, serverAutoConnect })` (only include `serverAutoConnect` if host non-empty), call `onUpdate` with result, call `onClose`
  6. Handle errors inline
  7. Backdrop click and Escape key call `onClose`
- **Verify:** `npx tsc -p tsconfig.renderer.json --noEmit` passes with 0 errors

### Task 3: Add edit button to InstanceList and wire modal
- **Type:** code
- **Files:** `src/renderer/components/instances/InstanceList.tsx`
- **Action:**
  1. Import `EditInstanceModal`
  2. Add state: `editingInstance: InstanceInfo | null` (null = modal closed)
  3. Add an edit button (`✎`) to each instance card between the info and the play button — styled similarly to `instance-card__delete` (background none, subtle color, hover accent)
  4. Clicking edit sets `editingInstance` to that instance
  5. `handleUpdated(updated: InstanceInfo)`: replace the matching entry in `instances` state by id
  6. Render `<EditInstanceModal>` when `editingInstance !== null`, passing it `editingInstance`, `onClose={() => setEditingInstance(null)}`, `onUpdate={handleUpdated}`
- **Verify:** `npx tsc -p tsconfig.renderer.json --noEmit` passes with 0 errors

### Task 4: Add edit button CSS
- **Type:** code
- **Files:** `src/renderer/styles/globals.css`
- **Action:**
  1. Add `.instance-card__edit` after `.instance-card__delete`:
     - Same base style as `instance-card__delete` (background none, border none, 32x32, cursor pointer)
     - Color `--text-muted`, hover color `--accent`, font-size 14px, border-radius `--radius-sm`
     - Transition color 0.15s
- **Verify:** Visual check

### Task 5: Verify both tsconfigs compile cleanly
- **Type:** verify
- **Action:** Run `npx tsc -p tsconfig.main.json --noEmit` and `npx tsc -p tsconfig.renderer.json --noEmit`
- **Verify:** Both pass with 0 errors
