    Apply all fixes identified in Task 1. Key items that are known to need attention:

    INSTANCES — add "Instance Types" subsection after the intro paragraph, before
    "Creating an Instance":

    ### Instance Types

    BlockHaven supports three instance types:

    **Server** — Connects to a remote Minecraft server. Optionally auto-connects on
    launch using `--quickPlayMultiplayer`. Add a server address in Create/Edit Instance
    to enable auto-connect. Server instances show a live ping status and MOTD on the
    instance card.

    **Singleplayer** — A fresh isolated environment for local single-player worlds.
    Minecraft launches to the main menu; create or open worlds normally. Worlds are
    stored in the instance's own game directory inside the BlockHaven data folder.

    **Imported** — Points BlockHaven at an existing Minecraft game directory on disk
    — from the official launcher (`~/.minecraft`), Prism Launcher, MultiMC, ATLauncher,
    or any other source. No files are moved or copied. Mods and shaders installed via
    BlockHaven go into the same directory. Important: avoid running two launchers
    against the same directory at the same time.

    INSTANCES — "Deleting an Instance":
    - If instance.ts delete() calls fs.rm (removes game files), update the text to say
      game files ARE deleted and warn the user. If it only removes the record, keep
      current text. Check the source and write accurately.

    INSTANCES — "Editing an Instance":
    - Confirm the edit button icon described matches the actual rendered icon (⚙ vs pencil)

    MODS — add Quilt to the mod loader mention wherever Fabric is called out alone.

    SESSION NOTES — ensure section heading, nav label reference ("Game Notes"), and

    SETTINGS — verify the documented settings list is complete and remove any that
    no longer exist. The BlockHaven default host/port settings exist but are only
    relevant for the auto-create flow that was removed — consider whether to keep or
    remove this setting doc entry based on whether the setting is still surfaced in UI.

    Do not add marketing language, filler, or excessive length. Help docs should be
    concise and factual.
