<objective>
Add backend support for the 'imported' instance type — a third instance variant that
points at an existing game directory on disk rather than creating a fresh isolated one.

Purpose: Lets users bring their existing Minecraft installations (default launcher,
Prism, MultiMC, ATLauncher) into BlockHaven without moving any files.

Output: 'imported' type in Instance model; instance creation skips mkdir for imported;
new instances:pick-directory IPC opens a native directory picker; preload and global
types updated to expose the new API.
</objective>

<task type="auto">
  <name>Task 2: Skip gameDirectory creation for imported instances</name>
  <files>src/core/game/instance.ts</files>
  <action>
    In InstanceManager.create(), the method currently always calls fs.mkdir for
    gameDirectory and its subdirs (mods, saves, etc.). For imported instances these
    directories already exist — don't create them.

    Change the mkdir block to be conditional:
      if (config.type !== 'imported') {
        await fs.mkdir(gameDirectory, { recursive: true });
        // ... subdirs
      }

    For imported instances, gameDirectory must be supplied in config (it's the path the
    user picked). Throw a clear error if config.type === 'imported' and no gameDirectory
    is provided:
      if (config.type === 'imported' && !config.gameDirectory) {
        throw new Error('gameDirectory is required for imported instances');
      }

    When type is imported, use config.gameDirectory directly instead of building a new
    path under getLauncherPaths().instances.

    Also pass type through to the stored Instance object (already done via
    `type: config.type ?? 'server'` from the previous phase — verify it's there).
  </action>
  <verify>pnpm tsc -p tsconfig.main.json --noEmit</verify>
  <done>Imported instances store the provided gameDirectory as-is; no mkdir called for imported type; error thrown if gameDirectory missing on imported</done>
</task>

<task type="auto">
  <name>Task 3: Add instances:pick-directory IPC handler and preload exposure</name>
  <files>src/main/ipc-handlers.ts, src/main/preload.ts</files>
  <action>
    In ipc-handlers.ts, add a new handler after the existing instances handlers:

      ipcMain.handle('instances:pick-directory', async () => {
        const win = BrowserWindow.getFocusedWindow();
        const result = await dialog.showOpenDialog(win ?? new BrowserWindow(), {
          title: 'Select Minecraft Game Directory',
          properties: ['openDirectory'],
          buttonLabel: 'Select Folder',
        });
        if (result.canceled || result.filePaths.length === 0) return null;
        return result.filePaths[0];
      });

    dialog is already imported from 'electron' at the top of ipc-handlers.ts — no new
    import needed.

    In preload.ts, add to the instances section:
      pickDirectory: () => ipcRenderer.invoke('instances:pick-directory'),
  </action>
