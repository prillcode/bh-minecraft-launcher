# 06-03 — Session Notes UI

## Objective
Build the Session Notes page: a two-pane journal where players record waypoints,
points of interest, and session memories for a multiplayer instance. Replaces
the removed My Worlds panel with a genuinely useful feature for server players.

## UI specification

### Layout
Two-pane layout inside `.notes` container:
- Left pane (`.notes__list`, ~280px fixed, full height, scrollable): note list + "New Note" button
- Right pane (`.notes__editor`, flex: 1, full height, scrollable): note editor or empty state

### Left pane — Note list
- "New Note" button at top (`.btn.btn--primary.btn--sm`, full width)
- Each note rendered as `.note-item` button:
  - Title (bold, truncated with ellipsis)
  - `updatedAt` formatted as relative time (reuse timeAgo helper pattern from removed WorldsPage)
  - Active note highlighted with `.note-item--active`
- Empty state text: "No notes yet. Click New Note to start."
- List sorted newest-first by `updatedAt`

### Right pane — Editor
- **No note selected / no instance selected**: centered empty state message
  - If no instance: "Select an instance to manage notes."
  - If instance selected but no note active: "Select a note or create a new one."
- **Note active**:
  - Editable title input (`.notes__title-input`, large font, no border styling — appears inline)
  - Textarea (`.notes__textarea`, flex: 1, resize: none, monospace-friendly)
  - Auto-save: debounce 500ms on title or text change, call `notes:update`
  - "Delete Note" button (`.btn.btn--danger.btn--sm`) in editor header row
  - **Screenshots section** below textarea:
    - Label "Screenshots"
    - Thumbnail grid (`.notes__screenshots`) — same `.world-card__thumb` sizing pattern
    - Each thumbnail: click opens lightbox overlay
    - "Browse Screenshots" button — calls `notes:listScreenshots(instanceId)`, opens a
      modal/picker overlay listing available screenshots from `gameDirectory/screenshots/`
      with checkboxes; confirm adds selected paths to `note.screenshotPaths` via `notes:update`
    - Per-thumbnail "×" remove button on hover

### New Note flow
1. User clicks "New Note"
2. Call `notes:create(instanceId, { title: 'New Note', text: '', screenshotPaths: [] })`
3. Prepend new entry to local list state, select it in editor
4. Focus title input

### Delete Note flow
1. User clicks "Delete Note"
2. Call `notes:delete(instanceId, entryId)`
3. Remove from local list, clear editor (select next note if any, else show empty state)
