# Plan: 05-01 ‚Äî Version Mismatch Warning (Backend)

**Phase:** 05 ‚Äî Version Mismatch Warning
**Work:** pcw-106 ux-improvements-v1

## Context

When an instance has `serverAutoConnect` set, the Minecraft client connects to the server immediately
on launch. If the client and server versions differ, Minecraft shows a disconnect screen ‚Äî not ideal UX.

**Desired behavior:**
- `game:launch` checks if `serverAutoConnect` is set AND `instance.serverMinecraftVersion` is set
- If the instance `versionId` differs from `serverMinecraftVersion`, return a structured warning
  to the renderer instead of launching
- Renderer shows a modal: "Server expects MC X.Y ‚Äî your instance is X.Z. Launch anyway?"
- User can proceed or cancel

**Why not auto-block?** The user may intentionally use a mismatched version (e.g. old snapshot testing).
A warning with proceed/cancel is the right pattern.

**`serverMinecraftVersion` source:**
- Stored on the instance: user sets it in EditInstanceModal (added in 05-02)
- Optionally: could also be auto-populated from `servers:ping` result when instance is selected
  (the ping already returns `version.name` ‚Äî but this is deferred to a later enhancement)

### Step 4 ‚Äî Add pre-launch version check to `game:launch`

**File:** `src/main/ipc-handlers.ts`

In the `game:launch` handler, after `const instance = await instanceManager.get(instanceId)`,
add:

```ts
// Version mismatch check
if (
  instance.serverAutoConnect &&
  instance.serverMinecraftVersion &&
  instance.serverMinecraftVersion !== instance.versionId
) {
  return {
    versionMismatch: true,
    instanceVersion: instance.versionId,
    serverVersion: instance.serverMinecraftVersion,
  };
}
```

This returns early with a structured object instead of launching.
The renderer (in 05-02) checks for `versionMismatch: true` and shows a warning dialog.
If the user proceeds, it calls a new `game:launch-force` channel (or reuses `game:launch`
with a `force` flag ‚Äî see below).

**Alternative (simpler): use a `force` parameter on `game:launch`:**

Change the handler signature to accept an optional second arg:
```ts
ipcMain.handle('game:launch', async (event, instanceId: string, force = false) => {
```

And gate the check:
```ts
if (!force && instance.serverAutoConnect && instance.serverMinecraftVersion && ...) {
  return { versionMismatch: true, ... };
}
```

The renderer calls `game:launch(instanceId, true)` when user clicks "Launch Anyway".
This avoids adding a second IPC channel.

**Use the `force` parameter approach.**
