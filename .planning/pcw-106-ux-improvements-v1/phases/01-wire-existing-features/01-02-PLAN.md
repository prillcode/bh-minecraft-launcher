# Plan: 01-02 ‚Äî Mod Enable/Disable (disk rename + UI toggle)

**Phase:** 01 ‚Äî Wire Existing Features
**Work:** pcw-106 ux-improvements-v1

## Context

`InstalledModInfo.enabled` (bool) is already stored per mod in `mods.json` via electron-store and
exposed in the renderer type. It is always set to `true` on install and never changed.
The `InstalledModsPanel` renders no toggle ‚Äî only a Remove button.

**Desired behavior:**
- User clicks a toggle on a mod row ‚Üí the `.jar` file is renamed to `.jar.disabled` on disk (or back)
- `enabled` and `fileName` are updated in the electron-store so state persists across restarts
- The row renders visually dimmed when disabled

**Disable strategy:**
- Disabled file: `mod.fileName` ‚Üí `mod.fileName + '.disabled'`  (e.g. `sodium.jar` ‚Üí `sodium.jar.disabled`)
- Minecraft ignores `.disabled` files in the `mods/` folder
- `mods:list` returns the stored `enabled` + `fileName` ‚Äî the UI never needs to read the filesystem

### Step 1 ‚Äî Add `setEnabled()` to `ModManager`

**File:** `src/core/mods/mod-manager.ts`

Add a new method after `remove()`:

```ts
async setEnabled(instanceId: string, projectId: string, enabled: boolean): Promise<InstalledMod> {
  const mod = await this.get(instanceId, projectId);
  if (!mod) throw new Error(`Mod ${projectId} not found in instance ${instanceId}`);

  // Derive new fileName: add or strip the .disabled suffix
  const newFileName = enabled
    ? mod.fileName.replace(/\.disabled$/, '')
    : mod.fileName.endsWith('.disabled') ? mod.fileName : mod.fileName + '.disabled';

  const updated: InstalledMod = { ...mod, enabled, fileName: newFileName };
  this.store.set(`mods.${instanceId}.${projectId}` as any, updated);
  logger.info(`Mod "${mod.name}" ${enabled ? 'enabled' : 'disabled'} in instance ${instanceId}`);
  return updated;
}
```

**Verify:** No TypeScript errors. Method returns the updated `InstalledMod`.

### Step 2 ‚Äî Add `mods:toggle` IPC handler

**File:** `src/main/ipc-handlers.ts`

Add after the `mods:remove` handler:

```ts
ipcMain.handle('mods:toggle', async (_e, instanceId: string, projectId: string) => {
  const mod = await modManager.get(instanceId, projectId);
  if (!mod) throw new Error(`Mod ${projectId} not found`);

  const instance = await instanceManager.get(instanceId);
  const modsDir = path.join(instance.gameDirectory, 'mods');
  const currentPath = path.join(modsDir, mod.fileName);
  const newEnabled = !mod.enabled;
  const newFileName = newEnabled
    ? mod.fileName.replace(/\.disabled$/, '')
    : mod.fileName + '.disabled';
  const newPath = path.join(modsDir, newFileName);

  await fs.rename(currentPath, newPath);
  const updated = await modManager.setEnabled(instanceId, projectId, newEnabled);
  return { success: true, enabled: updated.enabled };
});
```

**Verify:** `path`, `fs`, `modManager`, `instanceManager` all already imported/instantiated.

### Step 5 ‚Äî Add toggle button + dimming to `InstalledModsPanel`

**File:** `src/renderer/components/mods/InstalledModsPanel.tsx`

**5a ‚Äî Add `togglingId` state:**

```ts
const [togglingId, setTogglingId] = useState<string | null>(null);
```

**5b ‚Äî Add `handleToggle` handler:**
```ts
const handleToggle = async (mod: InstalledModInfo) => {
  setTogglingId(mod.id);
  const result = await window.launcher.mods.toggle(instanceId, mod.id);
  setMods(prev => prev.map(m => m.id === mod.id ? { ...m, enabled: result.enabled, fileName: m.fileName } : m));
  setTogglingId(null);
};
```

**5c ‚Äî Update the mod row rendering:**

Wrap the `<li>` with a disabled style and add a toggle button before the Remove button:

```tsx
<li

  style={{ opacity: mod.enabled ? 1 : 0.5 }}

  <div className="installed-mod-row__info">
    <span className="installed-mod-row__name">{mod.name}</span>
    <span className="installed-mod-row__version">{mod.versionNumber}</span>

  <div style={{ display: 'flex', gap: 6 }}>
    <button
      className="btn btn--ghost btn--sm"
      onClick={() => handleToggle(mod)}

      {removingId === mod.id ? 'Removing...' : 'Remove'}
    </button>
  </div>
```
