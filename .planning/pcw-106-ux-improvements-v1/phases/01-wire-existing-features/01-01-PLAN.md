# Plan: 01-01 — Wire closeOnLaunch + javaPath global override

**Phase:** 01 — Wire Existing Features
**Work:** pcw-106 ux-improvements-v1

## Context

Two settings are saved to disk but never read at runtime:

- `closeOnLaunch` (bool) — toggle in Settings > Game, default `false`
- `javaPath` (string) — text input in Settings > Java & Performance, default `''`

The `game:launch` IPC handler in `src/main/ipc-handlers.ts` is the entry point for both fixes.

**closeOnLaunch behavior expected:**
- When `true`: hide the launcher window after the game process spawns; re-show it when the game exits
- When `false`: do nothing (current behavior)

**javaPath behavior expected:**
- When non-empty: use this path as the Java executable instead of auto-detecting
- Priority: `instance.javaPath` > `settings.javaPath` > auto-detect
- Also: skip `javaProvisioner.provision()` when a global Java path is set (no need to provision managed JRE)

## Steps

### Step 1 — Add `globalJavaPath` to `LaunchOptions`

**File:** `src/core/game/types.ts`

Find the `LaunchOptions` interface. Add one optional field:

```ts
globalJavaPath?: string;   // from global settings, used when instance has no per-instance override
```

**Verify:** TypeScript compiles without errors after change.

---

### Step 2 — Use `globalJavaPath` in `GameLauncher.launch()`

**File:** `src/core/game/launch.ts`

Current line (approx line 37):
```ts
const javaPath = instance.javaPath ?? (await this.javaDetector.findBest(version));
```

Change to:
```ts
const javaPath =
  instance.javaPath ||
  options.globalJavaPath ||
  (await this.javaDetector.findBest(version));
```

No other changes to this file.

**Verify:** Logic reads instance override first, then global settings path, then auto-detects.

---

### Step 3 — Wire both settings in `game:launch` IPC handler

**File:** `src/main/ipc-handlers.ts`

**3a — Read settings at start of handler**

At the top of the `game:launch` handler body (after `const window = ...`), add:
```ts
const settings = getSettings();
```

**3b — Skip javaProvisioner when global javaPath is set**

Current check (approx line 165):
```ts
if (!instance.javaPath) {
  const component = version.javaVersion?.component ?? 'java-runtime-delta';
  await javaProvisioner.provision(component, ...);
}
```

Change condition to also skip when global path is set:
```ts
if (!instance.javaPath && !settings.javaPath) {
  const component = version.javaVersion?.component ?? 'java-runtime-delta';
  await javaProvisioner.provision(component, ...);
}
```

**3c — Pass `globalJavaPath` into `gameLauncher.launch()`**

In the `gameLauncher.launch({ ... })` call, add:
```ts
globalJavaPath: settings.javaPath || undefined,
```

**3d — Wire closeOnLaunch**

After the `gameLauncher.launch(...)` call resolves (i.e., after `const child = await gameLauncher.launch(...)`), add:

```ts
if (settings.closeOnLaunch) {
  window?.hide();
}
```

The existing `onExit` callback already sends `'launch:exit'` to the renderer. Extend it to also re-show the window:

```ts
onExit: (code) => {
  if (settings.closeOnLaunch) {
    window?.show();
  }
  window?.webContents.send('launch:exit', code);
},
```

**Verify:** `getSettings()` is already imported. `window` is already `BrowserWindow | null`.
