**src/core/game/instance.ts:**
Add a `migrateServerAutoConnect` method to the InstanceManager class. It takes a `ServerManager` instance (import the type lazily to avoid circular dep: `import type { ServerManager } from './server'`):

```ts
async migrateServerAutoConnect(serverManager: ServerManager): Promise<void> {
  const all = this.store.get('instances');
  for (const instance of Object.values(all)) {
    if (instance.serverAutoConnect && !instance.serverId) {
      const { host, port } = instance.serverAutoConnect;
      const server = await serverManager.create({
        name: instance.name,  // use instance name as the server display name
        host,
        port,
      });
      const updated = { ...instance, serverId: server.id };
      delete (updated as any).serverAutoConnect;
      this.store.set(`instances.${instance.id}`, updated);
      logger.info(`Migrated instance "${instance.name}" serverAutoConnect -> serverId ${server.id}`);
    }
  }
}
```

The migration is idempotent: instances with no serverAutoConnect or already with a serverId are skipped.

**src/main/ipc-handlers.ts:**
After both `instanceManager` and `serverManager` are instantiated, add the migration call before the `registerIpcHandlers` function body (or at the top of it):
```ts
// Migrate legacy serverAutoConnect -> serverId (idempotent)
instanceManager.migrateServerAutoConnect(serverManager).catch((err) =>
  logger.warn(`Server migration error: ${err}`)
);
```

Place this call at module level (right after the manager instantiations, not inside a handler).
