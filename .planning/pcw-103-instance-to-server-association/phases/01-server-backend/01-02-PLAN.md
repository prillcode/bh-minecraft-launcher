  <name>Task 1: Implement Server List Ping</name>
  <files>src/core/game/server-ping.ts</files>
  <action>
Create a class `ServerPing` with method `ping(host: string, port: number, timeoutMs = 5000): Promise<ServerPingResult>`.
Use Node.js `net` module only — no npm packages.

**Private static helpers:**

`encodeVarInt(value: number): Buffer`
  — encode int to varint (7 bits per byte, bit 7 = more bytes follow)

`readVarInt(buf: Buffer, offset: number): { value: number; bytesRead: number }`
  — decode varint from buffer at offset

`encodeString(str: string): Buffer`
  — UTF-8 bytes prefixed with varint of byte-length

**Handshake sequence:**
1. Create TCP socket, connect to host:port with `socket.setTimeout(timeoutMs)`.
2. Build Handshake packet payload:
   - `encodeVarInt(0)` (protocol version — 0 is fine for status queries)
   - `encodeString(host)`
   - 2-byte big-endian unsigned short for port (`buf.writeUInt16BE(port)`)
   - `encodeVarInt(1)` (next state = status)
3. Frame the packet: `[length_varint][packet_id_varint_0x00][payload]`
4. Send Status Request: `Buffer.from([0x01, 0x00])` (length=1, packet_id=0x00)
5. Accumulate incoming data until the full Status Response is parseable:
   - Parse length varint at offset 0
   - Parse packet ID varint (must be 0x00)
   - Parse JSON string: varint length then UTF-8 bytes
   - If buffer is too short, wait for more `data` events
6. Record `pingStart = Date.now()`, send Ping packet:
   `[length=9 varint][0x01 varint][8-byte payload (any bytes)]`
7. Wait for Pong response (same structure), compute `latencyMs = Date.now() - pingStart`
8. Destroy socket.

**Parse Status Response JSON** (shape: `{ version: { name: string }, players: { online: number, max: number }, description: string | { text: string } }`):
- `version` -> `version.name`
- `players.online` -> `onlinePlayers`, `players.max` -> `maxPlayers`
- MOTD from `description`: if string use directly; if object with `text` use `text`; strip `§` color codes with regex `/§[0-9a-fk-or]/gi`

**Error handling:**
- Wrap entire method in try/catch; on any error return `{ online: false, error: err.message }`
- Handle `socket.on('timeout', ...)` by destroying the socket (causes the catch to fire)
- Handle `socket.on('error', ...)` to reject cleanly

**Exports:**
- Export the `ServerPing` class
- Export a module-level `pingServer(host: string, port: number, timeoutMs?: number): Promise<ServerPingResult>` function that delegates to a singleton instance
